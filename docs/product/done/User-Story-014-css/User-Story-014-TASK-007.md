# User-Story-014-TASK-007: Manuelle Abnahme aller Änderungen im Browser

## Zugehörige Story
User-Story-014 – Einheitliches Erscheinungsbild der Benutzeroberfläche

## Beschreibung
Alle Änderungen aus TASK-001 bis TASK-006 sind im Browser manuell auf korrektes Erscheinungsbild und Fehlerfreiheit zu prüfen.

## Reihenfolge
Dieser Task wird als letzter ausgeführt, nachdem TASK-001 bis TASK-006 abgeschlossen sind.

## Prüfpunkte

| Nr. | Aktion | Erwartetes Ergebnis |
|---|---|---|
| 1 | Burger-Menü öffnen | Alle Einträge grau (`#6c757d`), kein Unterstrich |
| 2 | Hover über Burger-Menü-Eintrag | Eintrag wird weiß |
| 3 | POI-Marker anklicken | Modal-Titel schwarz, kein `text-primary`-Blau |
| 4 | Dialog „Geschichte" öffnen | Kein sichtbarer Rahmen um den Textinhalt |
| 5 | Dialog „Über das Projekt" öffnen | Kein sichtbarer Rahmen um den Textinhalt |
| 6 | Dialog „Features" öffnen | Kein sichtbarer Rahmen um den Textinhalt |
| 7 | Dialog „QR-Code" öffnen | Kein sichtbarer Rahmen um den Textinhalt |
| 8 | Dialog „Quellen" öffnen | Kein sichtbarer Rahmen um den Textinhalt |
| 9 | Dialog „Attribution" öffnen | Titel „Attribution" im Header sichtbar, kein Rahmen |
| 10 | Sprache wechseln (sofern mehrere konfiguriert, z. B. Moers) | Attribution-Titel bleibt „Attribution" |
| 11 | Browser-Konsole prüfen | Keine neuen JavaScript-Fehler |

## Schritte
- [ ] App im Browser starten
- [ ] Alle Prüfpunkte der Reihe nach durchgehen und abhaken
- [ ] Bei Abweichungen: zugehörigen Task erneut öffnen und korrigieren
